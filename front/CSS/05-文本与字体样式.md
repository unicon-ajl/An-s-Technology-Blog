
# 文本与字体样式

> ✒️ **排版是无声的沟通**。  
> 好的文本样式不仅美观，更能引导阅读、传递情绪、提升可读性。

---

## 字体（Font）基础

### 1. `font-family`：字体栈（Font Stack）
指定一组备选字体，浏览器从左到右尝试：
```css
body {
  font-family: "Helvetica Neue", Helvetica, Arial, sans-serif;
}
```
- **通用族名结尾**：`sans-serif`（无衬线）、`serif`（衬线）、`monospace`（等宽）；
- 系统字体更安全，避免依赖网络字体（除非必要）。

> 💡 推荐使用系统字体栈（性能好、加载快）：
> ```css
> /* macOS + Windows 通用 */
> font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, sans-serif;
> ```

---

### 2. `font-size`：字号
- **避免固定像素（px）用于正文** → 不利于用户缩放；
- **推荐相对单位**：
  - `em`：相对于父元素字体大小；
  - `rem`：相对于根元素（`<html>`）字体大小（✅ 更可预测）；
  - `1rem = 16px`（浏览器默认）。

```css
html { font-size: 18px; } /* 全局基准 */
h1 { font-size: 2.5rem; } /* = 45px */
p { font-size: 1rem; }    /* = 18px */
```

---

### 3. `font-weight`：字重
- 数值范围：`100`（极细）到 `900`（极粗），常用 `400`（正常）、`700`（粗体）；
- 并非所有字体都支持全部字重，需确认字体文件包含。

```css
.title { font-weight: 700; }
.subtitle { font-weight: 300; }
```

---

## 行高与对齐

### 1. `line-height`：行高 ✅ **影响可读性最关键属性**
- 无单位值（如 `1.6`）表示相对于当前字体大小的比例；
- 推荐值：正文 `1.5–1.8`，标题 `1.2–1.4`。

```css
p {
  font-size: 1rem;
  line-height: 1.6; /* 实际行高 = 1.6rem */
}
```
> 💡 行高过小 → 拥挤；过大 → 断裂感。

---

### 2. `text-align`：文本对齐
- `left` / `right` / `center` / `justify`（两端对齐）；
- **避免全文使用 `justify`** → 中文易出现不自然空隙。

---

### 3. `vertical-align`：垂直对齐（仅对 `inline`/`inline-block` 有效）
常用于图标与文字对齐：
```css
.icon {
  vertical-align: middle;
}
```
> ⚠️ 对块级元素无效！居中布局请用 Flexbox 或 Grid。

---

## 文本装饰与溢出

### 1. `text-decoration`：下划线、删除线等
```css
a { text-decoration: none; }           /* 去掉链接下划线 */
.strike { text-decoration: line-through; } /* 删除线 */
```

### 2. `text-transform`：大小写转换
```css
.upper { text-transform: uppercase; } /* 全大写 */
.capitalize { text-transform: capitalize; } /* 首字母大写 */
```

### 3. 处理长文本溢出
```css
.ellipsis {
  white-space: nowrap;      /* 禁止换行 */
  overflow: hidden;         /* 隐藏溢出 */
  text-overflow: ellipsis;  /* 显示省略号 */
}
```
> ✅ 必须配合 `overflow: hidden` 和固定宽度/最大宽度使用。

---

## 可访问性要点

1. **颜色对比度**：文本与背景的对比度 ≥ 4.5:1（WCAG 标准）；  
   → 工具：[WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/)
2. **不要仅靠颜色传递信息**：  
   ```html
   <!-- ❌ 错误 -->
   <span style="color: red;">错误</span>
   
   <!-- ✅ 正确 -->
   <span class="error">错误 ❌</span>
   ```
3. **允许用户放大文本**：避免用 `px` 锁死字号，确保布局在 200% 缩放下仍可用。

---

## 最佳实践

✅ 使用 `rem` 设置全局字号基准；  
✅ 正文行高 ≥ 1.5，提升阅读舒适度；  
✅ 长文本用 `text-overflow: ellipsis` 优雅截断；  
✅ 优先系统字体，兼顾性能与一致性；  
✅ 始终检查颜色对比度，保障无障碍。
