
# 什么是 CSS？

> 🎨 **CSS（Cascading Style Sheets）**，即“层叠样式表”，是用于**控制 HTML 元素外观**的样式语言。

它回答的问题很简单：  
> “这个标题该多大？这段文字什么颜色？按钮圆角多少？图片如何居中？”

但正是这些细节，决定了用户是“一眼爱上”还是“立刻离开”。

---

## 一个最简单的 CSS 示例

```html
<!DOCTYPE html>
<html>
<head>
  <style>
    h1 {
      color: navy;
      font-family: "Helvetica", sans-serif;
      text-align: center;
    }
  </style>
</head>
<body>
  <h1>欢迎来到我的网站</h1>
</body>
</html>
```

效果：  
- 标题变为深蓝色；  
- 使用无衬线字体；  
- 水平居中显示。

> 💡 注意：HTML 负责“这是个标题”，CSS 负责“标题长这样”。

---

## CSS 的基本结构：规则集（Rule Set）

每条 CSS 规则由三部分组成：

```css
selector {
  property: value;
}
```

- **`selector`（选择器）**：选中要样式化的 HTML 元素；
- **`property`（属性）**：要修改的样式特征（如颜色、大小）；
- **`value`（值）**：属性的具体设定。

### 示例解析
```css
p {
  color: #333;
  line-height: 1.6;
}
```
- 选中所有 `<p>` 段落；
- 设置文字颜色为深灰；
- 行高为 1.6 倍，提升可读性。

> ✅ 一条规则可包含多个声明，用分号 `;` 分隔。

## CSS 能做什么？

| 类别 | 能力 | 示例属性 |
|------|------|--------|
| **文本样式** | 字体、颜色、对齐 | `font-size`, `color`, `text-align` |
| **盒模型** | 间距、边框、尺寸 | `margin`, `padding`, `border`, `width` |
| **布局** | 定位、排列、对齐 | `display`, `position`, `flex`, `grid` |
| **视觉效果** | 背景、阴影、透明度 | `background`, `box-shadow`, `opacity` |
| **交互反馈** | 悬停、焦点状态 | `:hover`, `:focus` |
| **动画** | 过渡与关键帧动画 | `transition`, `@keyframes` |

---

## CSS 不能做什么？

❌ **不能改变 HTML 结构**  
→ 想加一个按钮？必须在 HTML 里写 `<button>`。

❌ **不能处理复杂逻辑**  
→ “如果用户登录就显示头像”？这需要 JavaScript。

❌ **不能保证跨浏览器完全一致**  
→ 但通过现代标准 + 合理降级，可达到高度一致。

> 📌 记住：**CSS 是表现层，不是逻辑层**。

## 层叠（Cascading）是什么意思？

“层叠”指当多个规则作用于同一元素时，**浏览器如何决定最终样式**。  
它基于三大原则：

1. **来源顺序**：后写的覆盖先写的；
2. **特异性（Specificity）**：ID > 类 > 元素；
3. **!important**：强制最高优先级（慎用！）。

### 示例
```css
/* 特异性低 */
p { color: blue; }

/* 特异性高 */
.warning { color: red; }
```
```html
<p class="warning">这段文字是红色</p>
```

> ⚠️ 滥用 `!important` 会导致样式难以维护，应优先通过合理选择器解决冲突。

## 常见误区

❌ “CSS 很简单，就是调颜色和大小”  
✅ 错！现代 CSS 包含布局引擎（Flex/Grid）、响应式系统、性能优化等复杂体系。

❌ “只要页面看起来对就行”  
✅ 错！忽略语义化类名、可访问性（如颜色对比度）、可维护性，后期代价巨大。
