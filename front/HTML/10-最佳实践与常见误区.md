
# 最佳实践与常见误区

> 🛠️ **写 HTML 很容易，写好 HTML 很难**。  
> 本篇总结高频陷阱与行业共识，助你写出专业、健壮、可维护的代码。

---

## ✅ 必须遵守的最佳实践

### 1. **始终声明 `<!DOCTYPE html>`**
```html
<!DOCTYPE html>
<html lang="zh-CN">
```
- 放在文件**第一行**；
- 确保浏览器使用**标准模式**渲染，避免“怪异模式”布局错乱。

---

### 2. **指定字符编码**
```html
<meta charset="UTF-8">
```
- 防止中文等非 ASCII 字符显示为乱码；
- **必须放在 `<head>` 顶部**（最好在 title 前）。

---

### 3. **所有属性值用引号包裹**
```html
<!-- ✅ 正确 -->
<div class="container" id="main"></div>

<!-- ❌ 错误（虽能运行，但不规范） -->
<div class=container id=main></div>
```
> 💡 引号避免空格、特殊字符解析错误，是 HTML5 规范推荐写法。

---

### 4. **自闭合标签不写斜杠**
```html
<!-- ✅ HTML5 推荐 -->
<img src="cat.jpg" alt="猫">
<br>
<hr>

<!-- ❌ XHTML 风格（HTML5 中多余） -->
<img src="cat.jpg" alt="猫" />
<br />
```
> 📌 注意：这仅适用于 **HTML 文档**（`.html`），XHTML 或 JSX 除外。

---

### 5. **每个页面只有一个 `<h1>`**
- `<h1>` 代表**当前页面主标题**（如文章名、产品名）；
- 多个 `<h1>` 会干扰 SEO 和屏幕阅读器导航。

---

### 6. **`<img>` 必须有 `alt` 属性**
- 信息图：`alt="2025 年前端趋势图"`
- 装饰图：`alt=""`（明确告知辅助技术可忽略）

> ⚠️ 缺失 `alt` 是无障碍重大缺陷。

---

## ❌ 高频误区与反模式

### 1. **用 `<br>` 模拟段落或间距**
```html
<!-- ❌ 错误 -->
第一段<br><br>
第二段

<!-- ✅ 正确 -->
<p>第一段</p>
<p>第二段</p>
```
> 💡 间距由 CSS 控制，语义由标签表达。

---

### 2. **滥用 `<div>` 和 `<span>`**
```html
<!-- ❌ 无语义 -->
<div class="button">提交</div>
<span class="title">欢迎</span>

<!-- ✅ 用原生语义标签 -->
<button type="submit">提交</button>
<h1>欢迎</h1>
```

---

### 3. **表格用于页面布局**
```html
<!-- ❌ 1990 年代做法 -->
<table>
  <tr><td>侧边栏</td><td>主内容</td></tr>
</table>
```
> ✅ 现代布局用 **CSS Flexbox 或 Grid**，表格只用于**展示数据**。

---

### 4. **忽略表单的 `<label>`**
```html
<!-- ❌ 无法通过点击文字聚焦 -->
<input type="checkbox"> 记住我

<!-- ✅ 关联 label -->
<label><input type="checkbox"> 记住我</label>
<!-- 或 -->
<input id="remember" type="checkbox">
<label for="remember">记住我</label>
```

---

### 5. **外部链接不加安全属性**
```html
<!-- ❌ 有安全风险 -->
<a href="https://example.com" target="_blank">外部链接</a>

<!-- ✅ 安全写法 -->
<a href="https..." target="_blank" rel="noopener noreferrer">外部链接</a>
```
> 🔒 `rel="noopener"` 防止新页面通过 `window.opener` 访问原页面。

## 🧪 开发建议

| 场景 | 推荐工具/方法 |
|------|--------------|
| **格式化** | 使用 Prettier 自动格式化 HTML |
| **验证** | [W3C Markup Validator](https://validator.w3.org/) |
| **检查 a11y** | 浏览器 Lighthouse、axe DevTools |
| **模板生成** | VS Code 输入 `!` + Tab 快速创建标准骨架 |

---

## 总结：好 HTML 的标准

- **结构清晰**：标签反映内容本质；
- **机器可读**：SEO 与无障碍友好；
- **人类易懂**：团队协作无歧义；
- **未来可维护**：不依赖 hack，遵循规范。

> ✅ 记住：**HTML 是内容的容器，不是样式的玩具**。
