
# 表格与表单基础

> 📊 ✍️ **表格用于展示结构化数据，表单用于收集用户输入**。  
> 它们是 Web 交互的核心组件，必须写得清晰、可用、无障碍。

---

## 表格：`<table>`

### 基本结构
```html
<table>
  <thead>
    <tr>
      <th>姓名</th>
      <th>年龄</th>
      <th>城市</th>
    </tr>
  </thead>
  <tbody>
    <tr>
      <td>小安</td>
      <td>28</td>
      <td>北京</td>
    </tr>
    <tr>
      <td>李雷</td>
      <td>32</td>
      <td>上海</td>
    </tr>
  </tbody>
</table>
```

### 关键标签说明
| 标签 | 作用 |
|------|------|
| `<table>` | 表格容器 |
| `<thead>` | 表头（可选但推荐） |
| `<tbody>` | 表体（可选但推荐） |
| `<tr>` | 表行（Table Row） |
| `<th>` | 表头单元格（加粗、居中） |
| `<td>` | 数据单元格 |

> 💡 提示：即使只有一行数据，也建议使用 `<thead>` + `<tbody>`，便于维护和样式控制。

---

### 无障碍增强：关联表头与数据
使用 `scope` 属性明确表头方向：
```html
<th scope="col">姓名</th>   <!-- 列表头 -->
<th scope="row">小安</th>   <!-- 行表头（如第一列为 ID） -->
```
> ✅ 屏幕阅读器会自动朗读：“姓名：小安”，提升理解效率。

---

## 表单：`<form>`

### 基本结构
```html
<form action="/submit" method="POST">
  <label for="email">邮箱：</label>
  <input type="email" id="email" name="email" required>

  <button type="submit">提交</button>
</form>
```

### 核心原则：**每个输入都应有 `<label>`**

- `for` 属性值 = `input` 的 `id`；
- 点击标签可聚焦输入框（提升移动端体验）；
- 屏幕阅读器依赖它识别字段用途。

> ❌ 错误：用 `<p>邮箱：</p><input>` → 无语义关联！

---

### 常用表单控件

| 控件 | 示例 | 说明 |
|------|------|------|
| 文本输入 | `<input type="text">` | 普通文本 |
| 邮箱 | `<input type="email">` | 自动校验格式 |
| 密码 | `<input type="password">` | 隐藏输入 |
| 复选框 | `<input type="checkbox">` | 多选 |
| 单选按钮 | `<input type="radio" name="gender">` | 同 `name` 为一组 |
| 下拉选择 | `<select><option>...</select>` | 枚举选项 |
| 提交按钮 | `<button type="submit">` | 触发表单提交 |

> 💡 优先使用语义化 `type`（如 `email`、`tel`），移动端会自动调出合适键盘。

---

### 表单无障碍要点

1. **必填字段标注**：
   ```html
   <label for="name">姓名 <span aria-label="（必填）">*</span></label>
   ```
2. **错误提示关联**：
   ```html
   <input aria-invalid="true" aria-describedby="error-email">
   <p id="error-email">请输入有效邮箱</p>
   ```

---

## 表格 vs 表单：别混淆！

| 用途 | 使用场景 |
|------|--------|
| **表格** | 展示数据（如成绩单、价格表） |
| **表单** | 收集数据（如登录、注册、调查） |

> ⚠️ 不要用 `<table>` 做表单布局！现代 CSS（Flex/Grid）才是正确方案。

---

## 总结

| 组件 | 关键实践 |
|------|--------|
| 表格 | 用 `<thead>`/`<tbody>`，`<th>` 加 `scope` |
| 表单 | 每个 `<input>` 配 `<label>`，用语义化 `type` |

> ✅ 好表格 = 数据清晰；好表单 = 输入顺畅 + 错误友好。
