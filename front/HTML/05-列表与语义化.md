
# 列表与语义化

> 📋 **列表是组织信息的天然方式**，而语义化标签让结构“会说话”。  
> 用对标签，机器能懂，用户受益。

---

## 三种标准列表

### 1. 无序列表 `<ul>`（Unordered List）
用于**无特定顺序**的项目：
```html
<ul>
  <li>苹果</li>
  <li>香蕉</li>
  <li>橙子</li>
</ul>
```
默认显示为带圆点的列表。

---

### 2. 有序列表 `<ol>`（Ordered List）
用于**有顺序或步骤**的内容：
```html
<ol>
  <li>打开编辑器</li>
  <li>新建 HTML 文件</li>
  <li>输入基础结构</li>
</ol>
```
默认显示为数字编号（1, 2, 3…）。

> 💡 可通过 `start` 或 `reversed` 属性控制起始值：
> ```html
> <ol start="5"> <!-- 从 5 开始 -->
> ```

---

### 3. 定义列表 `<dl>`（Description List）
用于**术语与解释**配对：
```html
<dl>
  <dt>HTML</dt>
  <dd>超文本标记语言，用于构建网页结构。</dd>
  
  <dt>CSS</dt>
  <dd>层叠样式表，用于控制页面外观。</dd>
</dl>
```
- `<dt>`：定义术语（Definition Term）  
- `<dd>`：描述内容（Definition Description）

> ✅ 适合词汇表、FAQ、元数据展示。

---

## 语义化容器：超越 `<div>`

HTML5 引入了**语义化区块标签**，替代泛滥的 `<div>`：

| 标签 | 用途 | 示例 |
|------|------|------|
| `<header>` | 页头（含 logo、导航等） | 网站顶部区域 |
| `<nav>` | 主要导航链接 | `<nav><a href="/home">首页</a></nav>` |
| `<main>` | 页面主内容（唯一） | 文章、产品详情 |
| `<article>` | 独立可分发内容 | 博客文章、新闻 |
| `<section>` | 主题分组区块 | 章节、功能模块 |
| `<aside>` | 侧边内容（非主干） | 广告、相关链接 |
| `<footer>` | 页脚信息 | 版权、联系方式 |

---

### 语义化结构示例
```html
<body>
  <header>
    <h1>小安的技术博客</h1>
    <nav>...</nav>
  </header>

  <main>
    <article>
      <h1>HTML 语义化指南</h1>
      <section>
        <h2>为什么重要？</h2>
        <p>...</p>
      </section>
    </article>
  </main>

  <aside>
    <h2>推荐阅读</h2>
    <ul>...</ul>
  </aside>

  <footer>
    <p>© 2025 小安</p>
  </footer>
</body>
```

> ✅ 这样的结构：
> - 屏幕阅读器可快速导航；
> - 搜索引擎更易理解内容层级；
> - 团队协作时代码意图一目了然。

---

## 常见误区

❌ 用 `<div>` 包裹导航 → ✅ 改用 `<nav>`  
❌ 整页只有一个 `<div class="content">` → ✅ 用 `<main>` + `<article>`  
❌ 把侧边栏叫 `<div class="sidebar">` → ✅ 用 `<aside>`

> 📌 提示：语义化不是“多此一举”，而是**为未来设计**。

---

## 如何验证语义是否合理？

问自己：
- “这个区块在没有样式时，还能看出作用吗？”
- “视障用户能通过标题/区域快速定位吗？”
- “搜索引擎能分清主次内容吗？”

如果答案是否定的，就该考虑换标签了。
