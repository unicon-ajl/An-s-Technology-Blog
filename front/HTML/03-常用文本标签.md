
# 常用文本标签

> ✍️ HTML 的核心任务之一：**准确表达文本的含义**。  
> 不是“怎么显示”，而是“这是什么”。

使用正确的标签，能让内容更清晰、更易访问、更利于 SEO。

---

## 标题：`<h1>` 到 `<h6>`

表示文档的层级结构：

```html
<h1>主标题（一篇文档只有一个）</h1>
<h2>章节标题</h2>
<h3>小节标题</h3>
<!-- ...最多到 h6 -->
```

### ✅ 正确用法
- 按层级顺序使用（不要跳级：`h1 → h3`）；
- `<h1>` 通常是页面主标题（如文章名）。

### ❌ 错误用法
```html
<!-- 为了更大字体而用 h1？错！ -->
<h1>这不是标题，只是普通文字</h1>

<!-- 用 CSS 控制大小，语义交给标签 -->
<p style="font-size: 2em;">这才是普通大字</p>
```

> 💡 屏幕阅读器用户依赖标题导航，乱用会让他们迷失。

---

## 段落：`<p>`

用于包裹**完整段落**：

```html
<p>HTML 是 Web 的结构语言。</p>
<p>它定义内容的含义，而非外观。</p>
```

> ⚠️ 不要用多个 `<br>` 模拟段落！  
> ❌ 错误：
> ```html
> 第一段<br><br>
> 第二段
> ```
> ✅ 正确：
> ```html
> <p>第一段</p>
> <p>第二段</p>
> ```

---

## 强调与重点

| 标签 | 含义 | 默认样式 |
|------|------|--------|
| `<strong>` | **重要内容**（语义强调） | 加粗 |
| `<em>` | *语气强调*（如重读） | 斜体 |

### 示例
```html
<p>请<strong>务必</strong>填写邮箱。</p>
<p>这个功能<em>可能</em>会有变化。</p>
```

### ❌ 避免过时写法
- `<b>`（仅视觉加粗，无语义）→ 改用 `<strong>`
- `<i>`（仅视觉斜体）→ 改用 `<em>` 或语义标签（如 `<cite>`）

> 📌 提示：在无障碍场景中，`<strong>` 和 `<em>` 会被语音合成器加重或变调。

---

## 换行与分隔线

| 标签 | 用途 | 建议 |
|------|------|------|
| `<br>` | 强制换行（如诗歌、地址） | 少用，避免布局依赖 |
| `<hr>` | 主题分隔线 | 语义化分隔，非装饰线 |

### 示例：地址
```html
<address>
  北京市朝阳区<br>
  某某科技园 A 座<br>
  邮编：100000
</address>
```

> ⚠️ 不要用 `<br>` 控制间距！用 CSS 的 `margin` 或 `padding`。

---

## 行内文本语义（进阶）

| 标签 | 用途 | 示例 |
|------|------|------|
| `<code>` | 代码片段 | `使用 <code>console.log()</code>` |
| `<mark>` | 高亮文本 | 搜索关键词 `<mark>HTML</mark>` |
| `<small>` | 法律声明、版权 | `<small>© 2025 小安</small>` |
| `<time>` | 时间/日期 | `<time datetime="2025-12-19">今天</time>` |

---

## 总结：选择标签的原则

1. **先问语义**：“这是标题？段落？强调？”
2. **再调样式**：用 CSS 控制外观；
3. **拒绝 `<div>` 泛滥**：能用语义标签就不用 `<div>`。

> ✅ 好 HTML = 内容 + 结构 + 含义，三位一体。
