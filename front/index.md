# 🌐 前端技术全景

欢迎来到「前端开发」模块！  
这里系统梳理了现代 Web 前端开发的核心技术栈，从基础到进阶，从原理到实践，助你构建清晰、扎实、可落地的知识体系。

---

## 🧱 三大基石

一切前端开发都建立在以下三项基础之上：

- **HTML**：页面结构的骨架，语义化是关键。
- **CSS**：视觉表现的画笔，掌握布局（Flex/Grid）与响应式设计是必备技能。
- **JavaScript**：交互逻辑的灵魂，理解作用域、闭包、异步、原型链是进阶起点。

> 💡 提示：别小看“基础”，90% 的性能与体验问题，根源都在这三者之中。

---

## ⚙️ 核心框架

主流 UI 框架让复杂应用开发变得高效可靠：

| 框架    | 特点                                                                 |
|--------|----------------------------------------------------------------------|
| **Vue** | 渐进式、上手快、生态完善，适合中小型项目快速交付。                     |
| **React** | 声明式、组件化强、社区庞大，适合大型应用与跨平台（React Native）。     |

我们深入探讨：
- 响应式原理（Vue 的 Proxy / React 的 Fiber）
- 组件通信与状态管理（Pinia / Redux / Zustand）
- 性能优化（虚拟滚动、懒加载、Memo）

---

## 🛠️ 工程化体系

现代前端 = 编码 + 构建 + 部署 + 监控

| 类别         | 关键工具                                                                 |
|--------------|--------------------------------------------------------------------------|
| **构建工具**   | [Vite](https://vitejs.dev/)（快！）、[Webpack](https://webpack.js.org/)（灵活） |
| **包管理**     | npm / Yarn / pnpm（速度与依赖扁平化）                                      |
| **代码规范**   | ESLint（语法检查） + Prettier（格式统一） + Husky（Git 钩子）               |
| **测试**       | Vitest / Jest（单元测试）、Cypress / Playwright（E2E）                    |

> ✨ 本博客基于 **Vite + VitePress** 构建，零配置启动，秒级热更新。

## 🎨 UI 与体验

好看 ≠ 花哨，好用才是王道：

- **UI 库**：Element Plus、Naive UI、Ant Design（按需引入，避免体积膨胀）
- **CSS 方案**：原生 CSS + BEM 命名 / Tailwind CSS（原子化） / CSS-in-JS（styled-components）
- **动效**：CSS 动画 / GSAP / Framer Motion（提升用户体验，但勿滥用）
- **无障碍（a11y）**：语义标签、ARIA 属性、键盘导航支持

---

## 📦 扩展能力

前端早已不止于浏览器：

- **PWA**：离线可用、消息推送、类原生体验
- **Web Components**：跨框架复用组件
- **微前端**：qiankun、Module Federation 实现多团队协作
- **低代码/可视化搭建**：JSON Schema + 渲染引擎

---

## 📚 学习建议

1. **先精后广**：深入一个框架（如 Vue），再横向对比（如 React）
2. **动手优先**：每学一个概念，立刻写 Demo 验证
3. **读源码**：从简单库（如 axios、mitt）开始，培养工程思维
4. **关注标准**：TC39、W3C、WHATWG 是技术演进的源头
