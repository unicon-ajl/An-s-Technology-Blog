面向对象编程（OOP） ​

🧱 面向对象不是“有 class 就是 OOP”，而是关于封装、复用与抽象的思维方式。
JavaScript 虽基于原型，但完全支持现代 OOP 的四大支柱：封装、抽象、继承、多态。

一、OOP 四大支柱在 JS 中的体现 ​

支柱含义JavaScript 实现方式
封装（Encapsulation）隐藏内部细节，暴露可控接口私有字段（#）、闭包、模块
抽象（Abstraction）忽略复杂细节，聚焦高层行为接口设计、高阶函数、类方法
继承（Inheritance）子类复用父类属性与方法原型链、extends
多态（Polymorphism）同一接口，不同实现方法重写、鸭子类型（Duck Typing）

💡 JavaScript 的 OOP 是基于原型的，而非传统“类-实例”模型，但 class 语法让其更贴近主流认知。

二、封装：隐藏内部状态 ​

1. 私有字段（ES2022+）✅ 现代标准 ​

js
class BankAccount {
  #balance = 0; // 私有字段，外部无法访问

  deposit(amount) {
    if (amount > 0) this.#balance += amount;
  }

  getBalance() {
    return this.#balance; // 通过公有方法暴露
  }
}

const account = new BankAccount();
account.deposit(100);
console.log(account.getBalance()); // 100
// console.log(account.#balance); // ❌ SyntaxError

✅ 私有字段以 # 开头，真正实现封装。

2. 闭包模拟私有（旧方案） ​

js
function createCounter() {
  let count = 0; // 外部不可访问
  return {
    increment: () => ++count,
    getCount: () => count
  };
}
  • 适用于函数式风格；
  • 但每个实例都创建新函数，内存开销大。

三、继承：代码复用的核心 ​

使用 class 和 extends ​

js
class Animal {
  constructor(name) {
    this.name = name;
  }

  speak() {
    console.log(`${this.name} makes a sound.`);
  }
}

class Dog extends Animal {
  constructor(name, breed) {
    super(name); // 调用父类构造函数
    this.breed = breed;
  }

  // 重写父类方法（多态）
  speak() {
    console.log(`${this.name} barks!`);
  }

  fetch() {
    console.log("Fetching the ball!");
  }
}

继承链验证 ​

js
const dog = new Dog("Rex", "Husky");
dog.speak(); // "Rex barks!"（多态）

console.log(dog instanceof Dog);     // true
console.log(dog instanceof Animal);  // true
console.log(dog instanceof Object);  // true

✅ 优先组合，慎用继承：若只是复用方法，考虑混入（Mixin）或工具函数。

四、多态：同一接口，多种行为 ​

多态在 JS 中天然存在——不依赖类型声明，只看对象是否有对应方法（鸭子类型）。

js
class Cat {
  speak() {
    console.log("Meow!");
  }
}

class Robot {
  speak() {
    console.log("Beep boop!");
  }
}

function makeSpeak(thing) {
  thing.speak(); // 只要 thing 有 speak 方法即可
}

makeSpeak(new Dog("Rex"));   // "Rex barks!"
makeSpeak(new Cat());        // "Meow!"
makeSpeak(new Robot());      // "Beep boop!"

💡 JavaScript 的多态比静态语言更灵活——无需继承同一基类。

五、抽象与接口（JS 中的模拟） ​

JavaScript 没有原生 interface，但可通过以下方式约定“契约”：

1. 抽象基类（抛出错误强制子类实现） ​

js
class Shape {
  area() {
    throw new Error("子类必须实现 area 方法");
  }
}

class Circle extends Shape {
  constructor(radius) {
    super();
    this.radius = radius;
  }

  area() {
    return Math.PI * this.radius ** 2;
  }
}

2. 文档 + TypeScript（推荐大型项目） ​

ts
// TypeScript 接口
interface Drawable {
  draw(): void;
}

class Button implements Drawable {
  draw() { /* ... */ }
}

✅ 纯 JS 项目可用 JSDoc 注释约定接口：

js
/**
 * @interface
 * @property {function(): void} draw - 绘制方法
 */

六、静态方法与属性 ​

属于类本身，而非实例：

js
class MathUtils {
  static PI = 3.14159;

  static randomInt(min, max) {
    return Math.floor(Math.random() * (max - min + 1)) + min;
  }
}

console.log(MathUtils.PI); // 3.14159
console.log(MathUtils.randomInt(1, 10)); // 如 7

✅ 用途：工具方法、工厂函数、常量配置。

七、getter / setter：控制属性访问 ​

js
class Temperature {
  constructor(celsius) {
    this._celsius = celsius;
  }

  get celsius() {
    return this._celsius;
  }

  set celsius(value) {
    if (value < -273.15) {
      throw new Error("温度不能低于绝对零度");
    }
    this._celsius = value;
  }

  get fahrenheit() {
    return this._celsius * 9/5 + 32;
  }
}

const temp = new Temperature(0);
console.log(temp.fahrenheit); // 32
temp.celsius = -300; // ❌ 抛出错误

✅ 用于数据校验、计算属性、兼容旧 API。

八、OOP 设计原则（SOLID 简化版） ​

  • 单一职责：一个类只做一件事；
  • 开闭原则：对扩展开放，对修改关闭；
  • 里氏替换：子类应能替换父类而不破坏逻辑；
  • 接口隔离：客户端不应依赖它不需要的接口；
  • 依赖倒置：依赖抽象，而非具体实现。

💡 在 JS 中，组合 > 继承 往往更灵活、更易测试。

最佳实践 ​

✅ 用 class + 私有字段（#）实现封装；
✅ 优先组合复用，避免深层继承树；
✅ 利用多态和鸭子类型编写通用函数；
✅ 用 static 定义工具方法，避免污染全局；
✅ 用 getter/setter 控制属性访问逻辑；
✅ 大型项目考虑 TypeScript 强化接口契约。

✅ 本篇亮点：

  • 明确 OOP 四大支柱在 JS 中的具体实现；
  • 展示私有字段（#）这一现代封装标准；
  • 强调“鸭子类型”使 JS 多态更灵活；
  • 提供抽象基类和 JSDoc 模拟接口的实用方案；
  • 融入 SOLID 原则，引导良好设计。
